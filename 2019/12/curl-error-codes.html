Curl Error Codes

2019-12-28

<p> CURLE_UNSUPPORTED_PROTOCOL (1) </p><p> The URL you passed to libcurl used a protocol that this libcurl does not support. The support might be a compile-time option that you didn't use, it can be a misspelled protocol string or just a protocol libcurl has no code for. </p><p> <a name="CURLEFAILEDINIT"></a> CURLE_FAILED_INIT (2) </p><p> Very early initialization code failed. This is likely to be an internal error or problem, or a resource problem where something fundamental couldn't get done at init time. </p><p> <a name="CURLEURLMALFORMAT"></a> CURLE_URL_MALFORMAT (3) </p><p> The URL was not properly formatted. </p><p> <a name="CURLENOTBUILTIN"></a> CURLE_NOT_BUILT_IN (4) </p><p> A requested feature, protocol or option was not found built-in in this libcurl due to a build-time decision. This means that a feature or option was not enabled or explicitly disabled when libcurl was built and in order to get it to function you have to get a rebuilt libcurl. </p><p> <a name="CURLECOULDNTRESOLVEPROXY"></a> CURLE_COULDNT_RESOLVE_PROXY (5) </p><p> Couldn't resolve proxy. The given proxy host could not be resolved. </p><p> <a name="CURLECOULDNTRESOLVEHOST"></a> CURLE_COULDNT_RESOLVE_HOST (6) </p><p> Couldn't resolve host. The given remote host was not resolved. </p><p> <a name="CURLECOULDNTCONNECT"></a> CURLE_COULDNT_CONNECT (7) </p><p> Failed to connect() to host or proxy. </p><p> <a name="CURLEWEIRDSERVERREPLY"></a> CURLE_WEIRD_SERVER_REPLY (8) </p><p> The server sent data libcurl couldn't parse. This error code was known as as CURLE_FTP_WEIRD_SERVER_REPLY before 7.51.0. </p><p> <a name="CURLEREMOTEACCESSDENIED"></a> CURLE_REMOTE_ACCESS_DENIED (9) </p><p> We were denied access to the resource given in the URL. For FTP, this occurs while trying to change to the remote directory. </p><p> <a name="CURLEFTPACCEPTFAILED"></a> CURLE_FTP_ACCEPT_FAILED (10) </p><p> While waiting for the server to connect back when an active FTP session is used, an error code was sent over the control connection or similar. </p><p> <a name="CURLEFTPWEIRDPASSREPLY"></a> CURLE_FTP_WEIRD_PASS_REPLY (11) </p><p> After having sent the FTP password to the server, libcurl expects a proper reply. This error code indicates that an unexpected code was returned. </p><p> <a name="CURLEFTPACCEPTTIMEOUT"></a> CURLE_FTP_ACCEPT_TIMEOUT (12) </p><p> During an active FTP session while waiting for the server to connect, the <a href="https://curl.haxx.se/libcurl/c/CURLOPT_ACCEPTTIMEOUT_MS.html" rel="noopener noreferer nofollow"> CURLOPT_ACCEPTTIMEOUT_MS </a> (or the internal default) timeout expired. </p><p> <a name="CURLEFTPWEIRDPASVREPLY"></a> CURLE_FTP_WEIRD_PASV_REPLY (13) </p><p> libcurl failed to get a sensible result back from the server as a response to either a PASV or a EPSV command. The server is flawed. </p><p> <a name="CURLEFTPWEIRD227FORMAT"></a> CURLE_FTP_WEIRD_227_FORMAT (14) </p><p> FTP servers return a 227-line as a response to a PASV command. If libcurl fails to parse that line, this return code is passed back. </p><p> <a name="CURLEFTPCANTGETHOST"></a> CURLE_FTP_CANT_GET_HOST (15) </p><p> An internal failure to lookup the host used for the new connection. </p><p> <a name="CURLEHTTP2"></a> CURLE_HTTP2 (16) </p><p> A problem was detected in the HTTP2 framing layer. This is somewhat generic and can be one out of several problems, see the error buffer for details. </p><p> <a name="CURLEFTPCOULDNTSETTYPE"></a> CURLE_FTP_COULDNT_SET_TYPE (17) </p><p> Received an error when trying to set the transfer mode to binary or ASCII. </p><p> <a name="CURLEPARTIALFILE"></a> CURLE_PARTIAL_FILE (18) </p><p> A file transfer was shorter or larger than expected. This happens when the server first reports an expected transfer size, and then delivers data that doesn't match the previously given size. </p><p> <a name="CURLEFTPCOULDNTRETRFILE"></a> CURLE_FTP_COULDNT_RETR_FILE (19) </p><p> This was either a weird reply to a 'RETR' command or a zero byte transfer complete. </p><p> <a name="CURLEQUOTEERROR"></a> CURLE_QUOTE_ERROR (21) </p><p> When sending custom "QUOTE" commands to the remote server, one of the commands returned an error code that was 400 or higher (for FTP) or otherwise indicated unsuccessful completion of the command. </p><p> <a name="CURLEHTTPRETURNEDERROR"></a> CURLE_HTTP_RETURNED_ERROR (22) </p><p> This is returned if <a href="https://curl.haxx.se/libcurl/c/CURLOPT_FAILONERROR.html" rel="noopener noreferer nofollow"> CURLOPT_FAILONERROR </a> is set TRUE and the HTTP server returns an error code that is >= 400. </p><p> <a name="CURLEWRITEERROR"></a> CURLE_WRITE_ERROR (23) </p><p> An error occurred when writing received data to a local file, or an error was returned to libcurl from a write callback. </p><p> <a name="CURLEUPLOADFAILED"></a> CURLE_UPLOAD_FAILED (25) </p><p> Failed starting the upload. For FTP, the server typically denied the STOR command. The error buffer usually contains the server's explanation for this. </p><p> <a name="CURLEREADERROR"></a> CURLE_READ_ERROR (26) </p><p> There was a problem reading a local file or an error returned by the read callback. </p><p> <a name="CURLEOUTOFMEMORY"></a> CURLE_OUT_OF_MEMORY (27) </p><p> A memory allocation request failed. This is serious badness and things are severely screwed up if this ever occurs. </p><p> <a name="CURLEOPERATIONTIMEDOUT"></a> CURLE_OPERATION_TIMEDOUT (28) </p><p> Operation timeout. The specified time-out period was reached according to the conditions. </p><p> <a name="CURLEFTPPORTFAILED"></a> CURLE_FTP_PORT_FAILED (30) </p><p> The FTP PORT command returned error. This mostly happens when you haven't specified a good enough address for libcurl to use. See <a href="https://curl.haxx.se/libcurl/c/CURLOPT_FTPPORT.html" rel="noopener noreferer nofollow"> CURLOPT_FTPPORT </a> . </p><p> <a name="CURLEFTPCOULDNTUSEREST"></a> CURLE_FTP_COULDNT_USE_REST (31) </p><p> The FTP REST command returned error. This should never happen if the server is sane. </p><p> <a name="CURLERANGEERROR"></a> CURLE_RANGE_ERROR (33) </p><p> The server does not support or accept range requests. </p><p> <a name="CURLEHTTPPOSTERROR"></a> CURLE_HTTP_POST_ERROR (34) </p><p> This is an odd error that mainly occurs due to internal confusion. </p><p> <a name="CURLESSLCONNECTERROR"></a> CURLE_SSL_CONNECT_ERROR (35) </p><p> A problem occurred somewhere in the SSL/TLS handshake. You really want the error buffer and read the message there as it pinpoints the problem slightly more. Could be certificates (file formats, paths, permissions), passwords, and others. </p><p> <a name="CURLEBADDOWNLOADRESUME"></a> CURLE_BAD_DOWNLOAD_RESUME (36) </p><p> The download could not be resumed because the specified offset was out of the file boundary. </p><p> <a name="CURLEFILECOULDNTREADFILE"></a> CURLE_FILE_COULDNT_READ_FILE (37) </p><p> A file given with FILE:// couldn't be opened. Most likely because the file path doesn't identify an existing file. Did you check file permissions? </p><p> <a name="CURLELDAPCANNOTBIND"></a> CURLE_LDAP_CANNOT_BIND (38) </p><p> LDAP cannot bind. LDAP bind operation failed. </p><p> <a name="CURLELDAPSEARCHFAILED"></a> CURLE_LDAP_SEARCH_FAILED (39) </p><p> LDAP search failed. </p><p> <a name="CURLEFUNCTIONNOTFOUND"></a> CURLE_FUNCTION_NOT_FOUND (41) </p><p> Function not found. A required zlib function was not found. </p><p> <a name="CURLEABORTEDBYCALLBACK"></a> CURLE_ABORTED_BY_CALLBACK (42) </p><p> Aborted by callback. A callback returned "abort" to libcurl. </p><p> <a name="CURLEBADFUNCTIONARGUMENT"></a> CURLE_BAD_FUNCTION_ARGUMENT (43) </p><p> Internal error. A function was called with a bad parameter. </p><p> <a name="CURLEINTERFACEFAILED"></a> CURLE_INTERFACE_FAILED (45) </p><p> Interface error. A specified outgoing interface could not be used. Set which interface to use for outgoing connections' source IP address with <a href="https://curl.haxx.se/libcurl/c/CURLOPT_INTERFACE.html" rel="noopener noreferer nofollow"> CURLOPT_INTERFACE </a> . </p><p> <a name="CURLETOOMANYREDIRECTS"></a> CURLE_TOO_MANY_REDIRECTS (47) </p><p> Too many redirects. When following redirects, libcurl hit the maximum amount. Set your limit with <a href="https://curl.haxx.se/libcurl/c/CURLOPT_MAXREDIRS.html" rel="noopener noreferer nofollow"> CURLOPT_MAXREDIRS </a> . </p><p> <a name="CURLEUNKNOWNOPTION"></a> CURLE_UNKNOWN_OPTION (48) </p><p> An option passed to libcurl is not recognized/known. Refer to the appropriate documentation. This is most likely a problem in the program that uses libcurl. The error buffer might contain more specific information about which exact option it concerns. </p><p> <a name="CURLETELNETOPTIONSYNTAX"></a> CURLE_TELNET_OPTION_SYNTAX (49) </p><p> A telnet option string was Illegally formatted. </p><p> <a name="CURLEGOTNOTHING"></a> CURLE_GOT_NOTHING (52) </p><p> Nothing was returned from the server, and under the circumstances, getting nothing is considered an error. </p><p> <a name="CURLESSLENGINENOTFOUND"></a> CURLE_SSL_ENGINE_NOTFOUND (53) </p><p> The specified crypto engine wasn't found. </p><p> <a name="CURLESSLENGINESETFAILED"></a> CURLE_SSL_ENGINE_SETFAILED (54) </p><p> Failed setting the selected SSL crypto engine as default! </p><p> <a name="CURLESENDERROR"></a> CURLE_SEND_ERROR (55) </p><p> Failed sending network data. </p><p> <a name="CURLERECVERROR"></a> CURLE_RECV_ERROR (56) </p><p> Failure with receiving network data. </p><p> <a name="CURLESSLCERTPROBLEM"></a> CURLE_SSL_CERTPROBLEM (58) </p><p> problem with the local client certificate. </p><p> <a name="CURLESSLCIPHER"></a> CURLE_SSL_CIPHER (59) </p><p> Couldn't use specified cipher. </p><p> <a name="CURLEPEERFAILEDVERIFICATION"></a> CURLE_PEER_FAILED_VERIFICATION (60) </p><p> The remote server's SSL certificate or SSH md5 fingerprint was deemed not OK. This error code has been unified with CURLE_SSL_CACERT since 7.62.0. Its previous value was 51. </p><p> <a name="CURLEBADCONTENTENCODING"></a> CURLE_BAD_CONTENT_ENCODING (61) </p><p> Unrecognized transfer encoding. </p><p> <a name="CURLELDAPINVALIDURL"></a> CURLE_LDAP_INVALID_URL (62) </p><p> Invalid LDAP URL. </p><p> <a name="CURLEFILESIZEEXCEEDED"></a> CURLE_FILESIZE_EXCEEDED (63) </p><p> Maximum file size exceeded. </p><p> <a name="CURLEUSESSLFAILED"></a> CURLE_USE_SSL_FAILED (64) </p><p> Requested FTP SSL level failed. </p><p> <a name="CURLESENDFAILREWIND"></a> CURLE_SEND_FAIL_REWIND (65) </p><p> When doing a send operation curl had to rewind the data to retransmit, but the rewinding operation failed. </p><p> <a name="CURLESSLENGINEINITFAILED"></a> CURLE_SSL_ENGINE_INITFAILED (66) </p><p> Initiating the SSL Engine failed. </p><p> <a name="CURLELOGINDENIED"></a> CURLE_LOGIN_DENIED (67) </p><p> The remote server denied curl to login (Added in 7.13.1) </p><p> <a name="CURLETFTPNOTFOUND"></a> CURLE_TFTP_NOTFOUND (68) </p><p> File not found on TFTP server. </p><p> <a name="CURLETFTPPERM"></a> CURLE_TFTP_PERM (69) </p><p> Permission problem on TFTP server. </p><p> <a name="CURLEREMOTEDISKFULL"></a> CURLE_REMOTE_DISK_FULL (70) </p><p> Out of disk space on the server. </p><p> <a name="CURLETFTPILLEGAL"></a> CURLE_TFTP_ILLEGAL (71) </p><p> Illegal TFTP operation. </p><p> <a name="CURLETFTPUNKNOWNID"></a> CURLE_TFTP_UNKNOWNID (72) </p><p> Unknown TFTP transfer ID. </p><p> <a name="CURLEREMOTEFILEEXISTS"></a> CURLE_REMOTE_FILE_EXISTS (73) </p><p> File already exists and will not be overwritten. </p><p> <a name="CURLETFTPNOSUCHUSER"></a> CURLE_TFTP_NOSUCHUSER (74) </p><p> This error should never be returned by a properly functioning TFTP server. </p><p> <a name="CURLECONVFAILED"></a> CURLE_CONV_FAILED (75) </p><p> Character conversion failed. </p><p> <a name="CURLECONVREQD"></a> CURLE_CONV_REQD (76) </p><p> Caller must register conversion callbacks. </p><p> <a name="CURLESSLCACERTBADFILE"></a> CURLE_SSL_CACERT_BADFILE (77) </p><p> Problem with reading the SSL CA cert (path? access rights?) </p><p> <a name="CURLEREMOTEFILENOTFOUND"></a> CURLE_REMOTE_FILE_NOT_FOUND (78) </p><p> The resource referenced in the URL does not exist. </p><p> <a name="CURLESSH"></a> CURLE_SSH (79) </p><p> An unspecified error occurred during the SSH session. </p><p> <a name="CURLESSLSHUTDOWNFAILED"></a> CURLE_SSL_SHUTDOWN_FAILED (80) </p><p> Failed to shut down the SSL connection. </p><p> <a name="CURLEAGAIN"></a> CURLE_AGAIN (81) </p><p> Socket is not ready for send/recv wait till it's ready and try again. This return code is only returned from <a href="https://curl.haxx.se/libcurl/c/curl_easy_recv.html" rel="noopener noreferer nofollow"> curl_easy_recv </a> and <a href="https://curl.haxx.se/libcurl/c/curl_easy_send.html" rel="noopener noreferer nofollow"> curl_easy_send </a> (Added in 7.18.2) </p><p> <a name="CURLESSLCRLBADFILE"></a> CURLE_SSL_CRL_BADFILE (82) </p><p> Failed to load CRL file (Added in 7.19.0) </p><p> <a name="CURLESSLISSUERERROR"></a> CURLE_SSL_ISSUER_ERROR (83) </p><p> Issuer check failed (Added in 7.19.0) </p><p> <a name="CURLEFTPPRETFAILED"></a> CURLE_FTP_PRET_FAILED (84) </p><p> The FTP server does not understand the PRET command at all or does not support the given argument. Be careful when using <a href="https://curl.haxx.se/libcurl/c/CURLOPT_CUSTOMREQUEST.html" rel="noopener noreferer nofollow"> CURLOPT_CUSTOMREQUEST </a> , a custom LIST command will be sent with PRET CMD before PASV as well. (Added in 7.20.0) </p><p> <a name="CURLERTSPCSEQERROR"></a> CURLE_RTSP_CSEQ_ERROR (85) </p><p> Mismatch of RTSP CSeq numbers. </p><p> <a name="CURLERTSPSESSIONERROR"></a> CURLE_RTSP_SESSION_ERROR (86) </p><p> Mismatch of RTSP Session Identifiers. </p><p> <a name="CURLEFTPBADFILELIST"></a> CURLE_FTP_BAD_FILE_LIST (87) </p><p> Unable to parse FTP file list (during FTP wildcard downloading). </p><p> <a name="CURLECHUNKFAILED"></a> CURLE_CHUNK_FAILED (88) </p><p> Chunk callback reported error. </p><p> <a name="CURLENOCONNECTIONAVAILABLE"></a> CURLE_NO_CONNECTION_AVAILABLE (89) </p><p> (For internal use only, will never be returned by libcurl) No connection available, the session will be queued. (added in 7.30.0) </p><p> <a name="CURLESSLPINNEDPUBKEYNOTMATCH"></a> CURLE_SSL_PINNEDPUBKEYNOTMATCH (90) </p><p> Failed to match the pinned key specified with <a href="https://curl.haxx.se/libcurl/c/CURLOPT_PINNEDPUBLICKEY.html" rel="noopener noreferer nofollow"> CURLOPT_PINNEDPUBLICKEY </a> . </p><p> <a name="CURLESSLINVALIDCERTSTATUS"></a> CURLE_SSL_INVALIDCERTSTATUS (91) </p><p> Status returned failure when asked with <a href="https://curl.haxx.se/libcurl/c/CURLOPT_SSL_VERIFYSTATUS.html" rel="noopener noreferer nofollow"> CURLOPT_SSL_VERIFYSTATUS </a> . </p><p> <a name="CURLEHTTP2STREAM"></a> CURLE_HTTP2_STREAM (92) </p><p> Stream error in the HTTP/2 framing layer. </p><p> <a name="CURLERECURSIVEAPICALL"></a> CURLE_RECURSIVE_API_CALL (93) </p><p> An API function was called from inside a callback. </p><p> <a name="CURLEAUTHERROR"></a> CURLE_AUTH_ERROR (94) </p><p> An authentication function returned an error. </p><p> <a name="CURLEHTTP3"></a> CURLE_HTTP3 (95) </p><p> A problem was detected in the HTTP/3 layer. This is somewhat generic and can be one out of several problems, see the error buffer for details. </p>
